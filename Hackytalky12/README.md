## github 협업 방식

1. 각자 브랜치를 판다.

```bash
git branch -M "브랜치 이름"
git checkout "브랜치 이름"
```

2. 수정사항이 있으면 push 한다

```bash
git add .
git commit -m "커밋 이름"
git push -u origin "브랜치 이름"
```

3. Pull request를 보낸다. 깃허브 레포 사이트 안에서 가능하다.
4. Main 브랜치와의 충돌 사항이 없으면 merge한다. 충돌 사항 있을 경우, main과 비교하여 적절히 조정한다.
